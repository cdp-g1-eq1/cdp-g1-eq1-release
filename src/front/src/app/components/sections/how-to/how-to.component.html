Documentation du site
Projects
Cette page regroupe les différents projets existants. Chaque projet est représenté par son nom et sa description. On peut voir « l’avancé » d’un projet (pourcentage de tasks réalisées) grâce à une barre de progression à droite de chaque projets.

Créer un nouveau projet :

- Cliquer sur le bouton « Nouveau projet » pour afficher le formulaire.

- Remplir les champs « Nom » et « Description » du formulaire (attention : deux projets ne peuvent avoir un même nom )

- Cliquer sur le bouton « Enregistrer » pour fermer le formulaire et ajouter le nouveau projet.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau projet.

Éditer un projet existant :

- Cliquer sur le bouton « Éditer » du projet correspondant pour afficher le formulaire pré-rempli de modification.

- Réaliser les modifications voulues (attention : le changement du nom du projet pour un nom déjà utilisé entraînera la non-validation des modifications)

- Cliquer sur le bouton « Enregistre » pour fermer le formulaire et appliquer les modifications réalisées.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau projet.

Supprimer un projet existant :

-Cliquer sur le bouton « Supprimer » du projet correspondant (attention : l’action est immédiate et irréversible)

Rechercher un projet existant :

- Entrer le(s) mot(s) clé(s) dans la barre de recherche.

- Cliquer sur le bouton « Rechercher » pour mettre à jour la liste des projets.

- Cliquer sur le bouton « Annuler la recherche » remettra la liste à son état initiale. (il n’est pas nécessaire de cliquer sur ce bouton pour réaliser une nouvelle recherche il suffit de changer la recherche dans la barre de recherche et cliquer sur le bouton « Rechercher »)

Accéder à un projet :

- Cliquer sur le titre du projet en question ouvrira le projet

- Cliquer sur le dernier onglet du menu à droite (onglet projets) permet de revenir en arrière.

Backlog

Cette section correspond au Backlog. C’est ici que vous trouverez les différentes User Stories du projet ainsi que leur répartition dans les différents sprints.

Créer une nouvelle User Story

- Cliquer sur le bouton « Ajouter une us » afin d’ouvrir le formulaire de création d’US.

- Remplir les différents champs du formulaire.

- Cliquer sur le bouton ajouter pour fermer le formulaire et ajouter la nouvelle US. Celle-ci sera par défaut placé dans le sprint « User Stories non planifiées »

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout de la nouvelle us.

Éditer une User Story existante :

- Cliquer sur le bouton « Éditer » de l’us correspondante pour afficher le formulaire pré-rempli de modification.

- Réaliser les modifications voulues.

- Cliquer sur le bouton «Éditer» pour fermer le formulaire et appliquer les modifications réalisées.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’édition du nouveau projet.

Déplacer une User Story vers un autre sprint :

- Réaliser un clique sur l’us souhaitée et maintenir celui-ci tout en déplaçant l’us vers le sprint choisi puis relâcher le clique.

- Relâcher l’us sur une zone non valide n’entraînera aucune action de déplacement de l’us.

Créer un nouveau sprint :
- Cliquer sur le bouton « Ajouter un sprint» afin d’ouvrir le formulaire de création de sprint.

- Renseigner le nom du nouveau sprint dans le formulaire.

- Cliquer sur le bouton ajouter pour fermer le formulaire et ajouter le nouveau sprint. Celui-ci sera par défaut placé au dessus du sprint « User Stories non planifiées »

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau sprint.

Démarrer un sprint :

-Cliquer sur le bouton « Démarrer » du sprint correspondant ce qui ouvrira la page sprint actif. (attention : une fois un sprint lancé il n’est plus possible de revenir en arrière. Son lancement entraînera sa fermeture totale et aucune action de modification ne sera plus possible.) Un seul sprint ne peut être démarré à la fois.

Tasks

Cette section contient toutes les tâches du projet. C’est dans cette section que l’on connaît l’état de chaque tâches. Il existe 3 états possibles pour une tâche : Todo, Doing et Done. Chaque tâche possède un label correspondant à l’état de son Dod (pas de Dod : « ! » , Dod en cours : « nb d’item validé sur nombre total » et Dod entièrement validé : « ok »)

Une estimation du temps de réalisation des tâches est présent et est actualisé à chaque changement (ajout/édition d’une tâche).

Ajouter une tâche :
- Cliquer sur le bouton « + » se trouvant à droite du titre de la section « Todo » pour afficher le formulaire de création.
- Compléter les informations du formulaire.
- Cliquer sur le bouton «Ajouter» pour fermer le formulaire et ajoute la tâche dans la section « Todo ».

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout de la nouvelle tâche.

Afficher les informations d’une tâche existante :

- Cliquer sur la tâche correspondante pour afficher une version complète des information d’une tâche.

- Cliquer sur le bouton « - » à côté d’une dépendance (tâche ou Us) permettra de supprimer celle-ci.

- Cliquer sur le bouton « + » après avoir fait un choix dans le menu déroulant des dépendances (tâche ou Us) permettra d’ajouter celle ci.

- Entrer du texte dans la partie « Definition of done » et cliquer sur le bouton « + » ajoutera une nouvelle ligne à la Dod.

Éditer une tâche existante :

- Cliquer sur la tâche correspondante.

- Cliquer sur le bouton « Modifier » pour afficher le formulaire.

- Modifier les informations du formulaire.

- Cliquer sur le bouton «Ajouter» pour fermer le formulaire et enregistrer la modification .

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans faire les modifications.

Déplacer une tâche :

- Réaliser un clique sur la tâche souhaitée et maintenir celui-ci tout en déplaçant la tâche vers la section choisi puis relâcher le clique. (attention : une tâche ne possédant pas de Dod ne peut être déplacée dans la section Doing et une tâche n’ayant pas tous les items de son Dod de validé ne pourra être déplacée dans la section Done)

- Relâcher la tâche sur une zone non valide n’entraînera aucune action de déplacement de la tâche.

Filtrer les tâches :

- Cliquer sur le bouton « filtrer les tâches » pour faire s’afficher le formulaire de filtre.

- Choisir le type de filtre pour faire afficher les options possible dans « Choix ».

- Cliquer sur « Filtrer » pour appliquer le filtre.

- Cliquer sur le bouton « Annuler Filtre » pour afficher toutes les tâches.

Planification

Dans cette section il vous sera possible de visualiser les dépendances des tâches du sprint actif ainsi qu’accéder à la gestion des membres du projet.

Ajouter un nouveau membre :

- Cliquer sur le bouton « Nouveau» afin d’ouvrir le formulaire de création de membre.

- Remplir les différents champs du formulaire.

- Cliquer sur le bouton ajouter pour fermer le formulaire et ajouter le nouveau membre.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau sprint.

Éditer un membre existant :

- Cliquer sur le nom du membre en question pour ouvrir le formulaire d’édition.

- Remplir les champs du formulaire.

- Cliquer sur le bouton « Enregistrer » pour fermer le formulaire et modifier le membre.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau sprint.

Accéder au graphique de dépendance :

- Cliquer sur le bouton « Démarrer » du sprint que vous voulez rendre actif.

- Aller dans la section planification.

- Déplacer les tâches en cliquant dessus et en effectuant en même temps des mouvements de déplacement avec la souris.

- Déplacer le graphique pour plus de visibilité en effectuant un clique sur une zone sans tâche (sans point de couleur) tout en bougeant la souris.

Sprint Actif

Dans cette section se trouve un récapitulatif du sprint actif. On peut y voir les us concernées et les tâches associées à celles-ci.

Accéder à la liste des tâches correspondantes :

-Cliquer sur l’User Story correspondante pour faire apparaître le nom des tâches associées à cette us.

- Cliquer à nouveau sur l’User Story pour faire disparaître les tâches.

Clôturer un sprint :

-Cliquer sur le bouton « Clôturer ». Une redirection vers Backlog est réalisée. Le sprint clôturé n’apparaît plus.

Tests

Dans cette section se trouve les différents tests du projet ainsi que leur état.

Ajouter un nouveau Test :

- Cliquer sur le bouton «Ajouter un test» afin d’ouvrir le formulaire de création de test.

- Remplir les différents champs du formulaire.

- Cliquer sur le bouton ajouter pour fermer le formulaire et ajouter le nouveau test.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau sprint.

Valider/Refuser un test :

-Cliquer sur le bouton « Valider » (« Refuser ») du test correspondant. Le liseré au dessus du test prend la couleur correspondante, la date de dernière exécution se met à jour et la jauge globale est actualisée. (attention : une fois un test validé/refusé il ne peut revenir à l’état neutre)

Éditer un test existant :

- Cliquer sur le bouton « éditer » du test en question pour ouvrir le formulaire d’édition.

- Modifier les champs du formulaire.

- Cliquer sur le bouton « Enregistrer » pour fermer le formulaire et modifier le test.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau test.

Supprimer un test existant :

-Cliquer sur le bouton « Supprimer » du test (attention : le test est automatiquement retiré de la liste, l’action est irréversible)

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout de la nouvelle release.

Supprimer une release existante :

-Cliquer sur le bouton « Supprimer » de la release (attention : la release est automatiquement retirée de la liste, l’action est irréversible)
